Overview - Problem Statement

This project focuses on industrial condition monitoring and predictive maintenance, aimed at building a robust vibration diagnostic network for high-value CNC machining centers. The core of the system is a custom Modbus RTU protocol implementation that allows for the simultaneous and reliable management of ten precision IMU sensors.

The development involved creating a scalable data acquisition network that monitors five distinct CNC machines (H1, H2, H3, H4, and STAR) in real-time. Each machine is equipped with two sensors: an "Internal" sensor on the mechanical unit and an "External" sensor on the chassis, enabling a comparative analysis of structural resonances versus machining vibrations. A primary challenge was ensuring reliable data collection at a central Master unit (PC) within an industrial environment where high-power motors generate significant electromagnetic interference (EMI). The solution utilized the RS485 physical layer combined with the Modbus RTU protocol to guarantee interference-free data transmission over long distances.

Development process - RS485 Physical Layer Preparation

The implementation began with the design and installation of the Physical Layer. Due to the high levels of EMI in the industrial setting, RS485 differential signaling was selected for its superior noise immunity. During the network configuration, each sensor was assigned a unique Modbus ID ranging from 0x50 to 0x59, ensuring precise addressing on the shared communication bus.

Critical aspects of the physical setup included specialized cabling and the strategic application of termination resistors to eliminate signal reflections. This hardware foundation provides the stability required for the Python-based Master software to poll each sensor node in the network at high-frequency 20ms intervals.

Development process - Modbus Programming

The software implementation focused on developing a library-independent, byte-level Modbus RTU protocol using Python. At the "heart" of the communication logic is a manually implemented CRC-16 (Cyclic Redundancy Check) algorithm. This algorithm utilizes the 0xA001 polynomial and bit-level shifting to provide a mathematical guarantee of data integrity. Requests are constructed using the struct.pack function with a ">B B H H" format, ensuring the strict Big-endian byte order required by the Modbus standard for control data.

On the receiving end, the software performs real-time CRC validation before parsing the response. Raw register data—such as acceleration values retrieved from address 0x34—is decoded and scaled into human-readable physical units, specifically G-force (g) and temperature (°C). To handle the high volume of incoming data efficiently, the system utilizes buffered logging: measurements are exported via the Pandas library to CSV files only after reaching a 10,000-row threshold. This approach minimizes disk I/O latency, ensuring uninterrupted data acquisition at a 9600 baud rate.
